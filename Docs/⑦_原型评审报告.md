## 1. 评审概述

### 1.1 评审目的

对 BeatSync 智能穿戴设备原型方案进行全面评审，评估技术可行性、成本合理性、风险可控性，给出「能否往下走」的明确结论。

### 1.2 评审范围

| 范围 | 内容 |
|------|------|
| 技术方案 | 步频检测、手势识别、骨传导播放三大核心技术路线 |
| 成本评估 | 原型 BOM 总成本 $79 |
| 竞品分析 | 与 Shokz/Garmin/Apple/Sony 的差异化定位 |
| 风险评估 | 音质、实时变速、结构可靠性三大风险 |
| 建议路线 | 分阶段验证方案 |

### 1.3 评审结论摘要

| 项目 | 结论 |
|------|------|
| 总体评价 | ✅ **可行，建议往下走** |
| 核心技术风险 | 🔴 骨传导音质为最大不确定性 |
| 成本评估 | $79 原型成本合理，物料易得 |
| 关键前提 | 必须先通过 Phase 1 核心验证（$50，2-3 周） |

## 2. 技术方案评审

### 2.1 步频检测

| 评估项 | 评价 |
|--------|------|
| 技术成熟度 | ✅ 高 — 峰值检测+带通滤波是成熟方案，开源参考丰富 |
| 算力匹配 | ✅ 充足 — ESP32C3 160MHz 单核完全胜任 |
| 精度预期 | ✅ 可达 ±2 BPM — 满足设计目标 |
| 实现难度 | 🟢 低 |
| **结论** | **无风险，可放心实现** |

### 2.2 手势识别

| 评估项 | 评价 |
|--------|------|
| 技术成熟度 | ✅ 中高 — DTW 在嵌入式平台有成熟实现 |
| 算力匹配 | ✅ 充足 — nRF52840 64MHz 可处理 DTW |
| 精度预期 | ✅ 可达 90%+ — 需采集训练数据，但 5 类手势可分性高 |
| 实现难度 | 🟡 中 — 需要模板采集流程 |
| **结论** | **可行，需预留模板采集时间** |

### 2.3 骨传导播放

| 评估项 | 评价 |
|--------|------|
| 技术成熟度 | ✅ 高 — DFPlayer + PAM8403 即插即用 |
| 音质预期 | 🔴 不确定 — 消费级解码 + 骨传导低频弱，可能仅「电话音质」 |
| 降级方案 | ✅ 已制定 — 相位声码器降级为步频匹配切歌 |
| 实现难度 | 🟢 低（链路搭建）/ 🔴 高（音质优化） |
| **结论** | **链路可行，音质为关键决策点** |

### 2.4 实时变速降级评估

| 评估项 | 评价 |
|--------|------|
| 原方案 | 相位声码器实时变速 — 单片机实现难度极高 |
| 降级方案 | 步频匹配切歌 — DFPlayer 原生支持 |
| 降级影响 | 功能从「实时变速」变为「切歌匹配」，用户体验有降级但核心价值保留 |
| **结论** | **降级方案合理，原型阶段接受** |

## 3. 成本评审

### 3.1 BOM 成本分析

| 端别 | 小计 | 占比 | 评价 |
|------|------|------|------|
| 耳机端 | $58 | 73.4% | 核心成本，模块化采购合理 |
| 戒指端 | $21 | 26.6% | 轻量化，成本可控 |
| **总计** | **$79** | 100% | ✅ 在 $80 目标内 |

### 3.2 成本结构分析

| 成本类别 | 金额 | 占比 | 备注 |
|---------|------|------|------|
| 主控芯片 | $13 | 16.5% | ESP32C3 + nRF52840 |
| 传感器 | $12 | 15.2% | 2× ICM-42688-P |
| 音频链路 | $16 | 20.3% | DFPlayer + PAM8403 + 振子 |
| 存储 | $6 | 7.6% | microSD 32GB |
| 电池+充电 | $12.5 | 15.8% | 2 电池 + TP4056 |
| 结构+杂项 | $19.5 | 24.7% | 3D 打印 + 线材 |

### 3.3 成本评价

- ✅ 总成本 $79 在 $80 预算内
- ✅ 所有物料为通用模块，采购渠道成熟（Seeed/淘宝/AliExpress）
- ✅ 单人可完成焊接组装，无外协成本
- ⚠️ 量产成本需重新评估（模块化方案不适合量产，需定制 PCB）

## 4. 竞品分析评审

### 4.1 差异化定位

BeatSync 的核心差异化组合：

| 差异点 | BeatSync | 现有竞品 |
|--------|----------|---------|
| 跑步时不用手机 | ✅ microSD 离线 | Shokz/Sony 需蓝牙连手机 |
| 手势控制无需抬手 | ✅ 戒指端手势 | 所有竞品需抬手或摸索按键 |
| 步频自动匹配 | ✅ 独有功能 | 无竞品有此功能 |

### 4.2 市场空间评估

- ✅ 「跑步时不用手机 + 手势控制 + 步频匹配」组合无直接竞品
- ⚠️ 细分市场规模需进一步调研
- ⚠️ 原型阶段无法评估品牌/渠道/售后等商业化因素

### 4.3 竞品分析评价

- ✅ 竞品对比维度全面，定位清晰
- ✅ 差异化亮点明确
- ⚠️ 需补充潜在竞品（如后续出现的跑步专用设备）

## 5. 风险评估

### 5.1 风险矩阵

| 编号 | 风险 | 严重程度 | 发生概率 | 风险等级 | 缓解措施 |
|------|------|---------|---------|---------|---------|
| R1 | 骨传导音质不达标 | 高 | 中 | 🔴 高 | Phase 1 优先验证；备选 I2S DAC + EQ |
| R2 | 实时变速实现困难 | 高 | 高 | 🔴 高 | 已降级为步频匹配切歌 |
| R3 | TPU 外壳疲劳断裂 | 中 | 高 | 🟡 中 | 接受有限寿命；后续钛合金丝骨架 |
| R4 | 手势识别准确率不足 | 中 | 低 | 🟡 中 | DTW 有成熟参考，预留模板采集时间 |
| R5 | BLE 连接不稳定 | 中 | 低 | 🟡 中 | nRF52840 BLE 成熟，短距离可靠 |
| R6 | 戒指端续航不足 | 低 | 中 | 🟢 低 | 50mAh 待机 8h 可达成，IMU 功耗低 |
| R7 | microSD 兼容性问题 | 低 | 低 | 🟢 低 | DFPlayer 对 FAT32 兼容性成熟 |

### 5.2 关键风险详评

#### R1: 骨传导音质（🔴 最高风险）

| 评估项 | 内容 |
|--------|------|
| 风险描述 | DFPlayer Mini 消费级解码 + 骨传导振子低频天然弱，最终音质可能仅为「电话音质」 |
| 影响 | 严重影响用户体验，动摇产品核心差异化基础 |
| 概率 | 中（50%）— 骨传导音质本身有天花板，但「可接受」标准因人而异 |
| 缓解措施 | Phase 1 第一优先验证；备选 I2S DAC (PCM5102) + EQ 低频补偿 |
| 止损条件 | 如果骨传导声音完全不能接受 → 建议及时止损 |
| **评审意见** | **音质是项目的「生死线」，Phase 1 必须给出明确结论** |

#### R2: 实时变速（🔴 已降级）

| 评估项 | 内容 |
|--------|------|
| 原风险 | 相位声码器在单片机上做实时变速是独立项目级难度 |
| 降级方案 | 改为「步频匹配切歌」— 检测到步频变化切换 BPM 更匹配的歌 |
| 降级评价 | ✅ 合理 — 保留核心价值（步频与音乐同步），实现难度大幅降低 |
| **评审意见** | **降级方案接受，原型阶段不追求实时变速** |

#### R3: TPU 外壳可靠性（🟡 中风险）

| 评估项 | 内容 |
|--------|------|
| 风险描述 | 3D 打印层间附着力在反复弯折后疲劳开裂，2-3 个月可能断裂 |
| 影响 | 日常使用耐久性问题，不影响功能验证 |
| 缓解措施 | 原型阶段接受有限寿命；后续嵌入钛合金丝做骨架 |
| **评审意见** | **原型阶段可接受，量产前需解决** |

## 6. 建议路线评审

### 6.1 分阶段方案

| 阶段 | 内容 | 预算 | 周期 | 评审意见 |
|------|------|------|------|---------|
| Phase 1 | 核心验证（面包板） | $50 | 2-3 周 | ✅ 合理，先验证最大风险 |
| Phase 2 | 完整原型（含戒指+外壳） | +$30 | 2-3 周 | ✅ Phase 1 通过后执行 |
| 决策点 | 骨传导音质不可接受 → 止损 | — | — | ✅ 关键决策点设置合理 |

### 6.2 Phase 1 验证内容

| 验证项 | 方法 | 通过标准 | 评审意见 |
|--------|------|---------|---------|
| 步频检测精度 | 跑步机对比 Garmin | 误差 ≤ ±2 BPM | ✅ 可验证 |
| 骨传导音质 | 主观评估 1~5 分 | 可接受度 ≥ 3 分 | ✅ 可验证 |
| DFPlayer 链路 | 播放/切歌/音量 | 功能正常 | ✅ 可验证 |

### 6.3 评审意见

- ✅ 分阶段方案合理，先验证最大风险（音质）再投入完整原型
- ✅ $50 核心验证预算合理，时间 2-3 周可行
- ✅ 止损决策点设置正确 — 音质是核心差异化基础，不可妥协
- ⚠️ 建议 Phase 1 同时验证 BLE 基本连通性（耳机端 ↔ 戒指端）
