## 3. BLE 通信协议

### 3.1 连接参数

| 参数 | 值 |
|------|-----|
| 角色（耳机端） | Central / Master |
| 角色（戒指端） | Peripheral / Slave |
| 广播间隔 | 100ms |
| 连接间隔 | 20ms ~ 50ms |
| 超时 | 5 秒 |
| 安全 | 无加密（原型阶段） |

### 3.2 GATT Service 定义

#### Service: BeatSync Gesture Service

| 字段 | 值 |
|------|-----|
| UUID | 0xFFB0 (自定义) |
| 名称 | BeatSync Gesture Service |

#### Characteristic: Gesture Data

| 字段 | 值 |
|------|-----|
| UUID | 0xFFB1 |
| 属性 | Notify |
| 数据长度 | 3 字节 |
| 通知间隔 | 事件触发 |

**数据格式**:

| 字节 | 内容 | 说明 |
|------|------|------|
| Byte 0 | Gesture ID | 0x01=单击, 0x02=双击, 0x03=上滑, 0x04=下滑, 0x05=旋转 |
| Byte 1 | Confidence | 0~100, DTW 匹配置信度 |
| Byte 2 | Reserved | 保留，0x00 |

#### Characteristic: Battery Level

| 字段 | 值 |
|------|-----|
| UUID | 0xFFB2 |
| 属性 | Read + Notify |
| 数据长度 | 1 字节 |
| 通知间隔 | 电量变化 5% 时 |

**数据格式**: 0~100，电池百分比

#### Characteristic: Heartbeat

| 字段 | 值 |
|------|-----|
| UUID | 0xFFB3 |
| 属性 | Write |
| 数据长度 | 1 字节 |
| 写入间隔 | 2 秒 |

**数据格式**: 0xAA（心跳包）

### 3.3 通信流程

```
1. 戒指端上电 → 开始 BLE 广播
2. 耳机端扫描 → 发现戒指端 → 建立连接
3. 耳机端订阅 Gesture Notify + Battery Notify
4. 戒指端检测到手势 → 发送 Gesture Notification
5. 耳机端收到手势 → 执行对应操作
6. 耳机端每 2 秒写入 Heartbeat → 戒指端回复（保活）
7. 超时 5 秒未收到心跳 → 戒指端断开重连
```

## 5. 电源架构

### 5.1 耳机端电源

```
USB 5V ──► TP4056 ──► 1200mAh 锂电池 (3.7V)
                         │
                    TP4056 OUT (5V)
                         │
               ┌─────────┼─────────┐
               │         │         │
          DFPlayer   PAM8403   XIAO 5V
               │                   │
               │              XIAO 3V3 (板载LDO)
               │                   │
               │              ICM-42688-P
               │
          (骨传导振子)
```

### 5.2 戒指端电源

```
50mAh 锂电池 (3.7V)
       │
  XIAO nRF52840 BAT+
       │
  XIAO 3V3 (板载LDO)
       │
  ICM-42688-P VDD
```
