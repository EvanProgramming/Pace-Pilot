## 1. 项目概述

### 1.1 产品概念

BeatSync 是一款面向跑步场景的智能穿戴设备，由**骨传导耳机端**和**智能戒指端**两部分组成。核心创新点：

1. **手指手势控制音乐** — 通过戒指端 IMU 检测手指动作，无需抬手即可切歌/调音量
2. **步频自动匹配切歌** — 耳机端实时检测跑步步频，自动切换至 BPM 匹配的曲目
3. **完全离线播放** — microSD 本地存储，跑步时无需携带手机

### 1.2 适用场景

- 户外跑步/室内跑步机训练
- 用户希望开放双耳感知环境声音
- 用户不想携带手机但仍需音乐陪伴

### 1.3 设计目标

| 指标 | 目标值 |
|------|--------|
| 步频检测精度 | 误差 ≤ ±2 BPM |
| 手势识别准确率 | ≥ 90%（5 类手势） |
| 续航（耳机端） | ≥ 4 小时连续播放 |
| 续航（戒指端） | ≥ 8 小时待机 |
| 骨传导音质 | 可接受（不低于电话音质） |
| 原型总成本 | ≤ $80 |

## 2. 系统架构

### 2.1 整体拓扑

```
┌─────────────────────────────────────────────────────────┐
│                    BeatSync 系统拓扑                      │
│                                                         │
│  ┌──────────────┐         BLE          ┌──────────────┐ │
│  │   戒指端      │◄───────────────────►│   耳机端      │ │
│  │              │   手势数据上行        │              │ │
│  │ XIAO nRF52840│                      │ XIAO ESP32C3 │ │
│  │ ICM-42688-P  │                      │ ICM-42688-P  │ │
│  │ 50mAh 电池    │                      │ DFPlayer Mini│ │
│  │ 3D打印TPU外壳 │                      │ PAM8403 功放  │ │
│  └──────────────┘                      │ 骨传导振子×2  │ │
│                                        │ microSD 32GB │ │
│                                        │ 1200mAh 电池  │ │
│                                        │ TP4056 充电   │ │
│                                        │ 按键×3        │ │
│                                        └──────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 2.2 耳机端架构

耳机端是系统的核心，承担以下职责：

| 功能模块 | 负责内容 | 关键器件 |
|---------|---------|---------|
| 主控 | 系统调度、步频算法、BLE 通信 | Seeed XIAO ESP32C3 |
| 运动感知 | 步频检测、加速度采集 | ICM-42688-P |
| 音频解码 | microSD 读取 + MP3/WAV 解码 | DFPlayer Mini |
| 音频放大 | 驱动骨传导振子 | PAM8403 |
| 骨传导发声 | 将电信号转为振动 | 14mm 骨传导振子 ×2 |
| 本地存储 | 音乐文件存储 | microSD 32GB (FAT32) |
| 供电管理 | 锂电池充放电保护 | 1200mAh + TP4056 |
| 用户交互 | 模式切换/音量调节 | 轻触开关 ×3 |

### 2.3 戒指端架构

戒指端是辅助输入设备，职责单一：

| 功能模块 | 负责内容 | 关键器件 |
|---------|---------|---------|
| 主控 | 手势识别算法、BLE 从机 | Seeed XIAO nRF52840 |
| 运动感知 | 手指动作采集 | ICM-42688-P |
| 供电 | 小型锂电池 | 50mAh |
| 外壳 | 戒指形态佩戴 | 3D 打印 TPU |

### 2.4 通信架构

```
耳机端 (BLE Central)          戒指端 (BLE Peripheral)
    │                               │
    │  ── BLE GATT 连接 ──────────► │
    │                               │
    │  ◄── 手势数据通知 ───────────  │
    │      (Gesture UUID)           │
    │                               │
    │  ── 心跳/状态查询 ──────────► │
    │  ◄── 电量/状态响应 ──────────  │
    │                               │
```

- **协议**: BLE 4.2
- **连接模式**: 戒指端为 Peripheral，耳机端为 Central
- **数据通道**: 自定义 GATT Service，手势事件通过 Notification 上行

## 3. 技术路线

### 3.1 步频检测

| 项目 | 说明 |
|------|------|
| 传感器 | ICM-42688-P 加速度计，采样率 100Hz |
| 算法 | 峰值检测 + 滑动窗口平均 |
| 算力评估 | ESP32C3 (160MHz) 算力充足，单核即可处理 |
| 风险等级 | 🟢 低 — 成熟算法，开源参考丰富 |
| 输出 | 实时步频 BPM（更新频率 1Hz） |

**算法流程**:
1. 读取 Z 轴加速度数据（垂直方向）
2. 带通滤波（0.5Hz ~ 5Hz），去除重力分量和高频噪声
3. 滑动窗口（2 秒）内峰值检测
4. 峰值间隔 → 步频 BPM
5. 1Hz 更新频率输出至主状态机

### 3.2 手势识别

| 项目 | 说明 |
|------|------|
| 传感器 | ICM-42688-P 6 轴（加速度+陀螺仪），采样率 100Hz |
| 算法 | DTW（动态时间规整）模板匹配 |
| 手势类别 | 5 类：单击、双击、上滑、下滑、旋转 |
| 目标准确率 | ≥ 90% |
| 风险等级 | 🟡 中 — 有开源参考，需采集训练数据 |
| 参考 | DTW 手势识别在嵌入式平台有成熟实现 |

**识别流程**:
1. 戒指端 IMU 持续采集 6 轴数据
2. 能量阈值检测触发手势录制窗口（1.5 秒）
3. 6 轴时序数据通过 DTW 与预存模板比对
4. 最小距离对应手势类型 → BLE 上行至耳机端
5. 耳机端执行对应操作（切歌/调音量/模式切换）

**手势映射**:

| 手势 | 功能 |
|------|------|
| 单击 | 播放/暂停 |
| 双击 | 下一首 |
| 上滑 | 音量+ |
| 下滑 | 音量- |
| 旋转 | 切换步频匹配模式 开/关 |

### 3.3 骨传导播放

| 项目 | 说明 |
|------|------|
| 解码方案 | DFPlayer Mini，硬件解码 MP3/WAV |
| 存储介质 | microSD 32GB，FAT32 文件系统 |
| 功放 | PAM8403 D 类功放，3W×2 |
| 发声 | 14mm 骨传导振子 ×2，8Ω |
| 风险等级 | 🔴 高 — 音质是最大不确定性 |
| 降级方案 | 原型阶段使用「步频匹配切歌」替代「实时变速」 |

**音频链路**:
```
microSD → DFPlayer Mini (解码) → PAM8403 (功放) → 骨传导振子×2 (发声)
               ↑
          ESP32C3 (UART 控制)
```

**步频匹配切歌逻辑**:
1. 维护歌曲库，每首歌预先标注 BPM
2. 实时步频与当前歌曲 BPM 差值超过阈值（±10 BPM）持续 10 秒
3. 从歌库中选取 BPM 最接近当前步频的下一首
4. 通过 DFPlayer UART 指令切歌

## 5. 可行性分析

### 5.1 可行性结论

**原型阶段：✅ 可行**

核心功能在模块层面均已验证，物料便宜易得，单人可完成焊接组装。

### 5.2 关键技术验证点

| 验证点 | 状态 | 说明 |
|--------|------|------|
| 步频检测 | ✅ 成熟 | 成熟算法 + ESP32C3 算力充足，风险低 |
| 手势识别 | ✅ 可行 | DTW 模板匹配有开源参考，5 类手势 92% 准确率可达成 |
| 骨传导播放 | ✅ 即插即用 | DFPlayer + PAM8403 链路即插即用 |

### 5.3 风险评估

| 等级 | 风险 | 影响 | 缓解措施 |
|------|------|------|---------|
| 🔴 高 | 骨传导音质 | DFPlayer Mini 消费级解码 + 骨传导振子低频弱，最终音质可能仅为「电话音质」 | 原型阶段先验证可接受度；若不可接受，升级为 I2S DAC + EQ 补偿方案 |
| 🔴 高 | 实时变速复杂性 | 相位声码器在单片机实时变速是独立项目级难度 | 已降级为「步频匹配切歌」，DFPlayer 原生支持 |
| 🟡 中 | TPU 后挂长期可靠性 | 3D 打印层间附着力在反复弯折后疲劳开裂，可能 2-3 个月断裂 | 原型阶段接受有限寿命；后续可嵌入钛合金丝做骨架 |
